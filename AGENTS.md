# 数学与运筹科研 Agents 工作台指令

本项目是偏数学与运筹学的科研工作台。默认使用中文沟通、中文产物与中文目录；Lean、Sage、LaTeX 内部工程文件可使用英文名以保证工具兼容。

## 主脑与技能路由

- 默认把 Codex GPT-5.5 xhigh 作为主脑。
- 外层科研流程优先使用 EvoSkills：`paper-navigator`、`research-survey`、`paper-planning`、`paper-review`、`evo-memory`。
- 数学推导优先使用本机技能 `formula-derivation`。
- 定理、命题、引理、证明补全优先使用本机技能 `proof-writer`。
- 论文级审查优先使用 `paper-review`，但数学正确性必须先过本项目的门禁。

## 核心闭环

任何核心研究结果都必须闭环到：

```text
建模对象 -> 数学命题 -> 可解优化形式/算法 -> 论文 claim
```

不得把聊天中的灵感直接写成论文结论。每个强 claim 必须在 `00_总览索引/claim审批表.md` 或对应命题目录中登记状态。

## Auto-Gate 常用入口

基础薄弱用户不需要先判断任务类型，也不需要亲自承担数学 PI 终审。可以直接使用以下短指令触发默认流程：

- `Auto-Gate 接管这轮输入。`
- `处理这份材料。`
- `审计这篇文献。`
- `检查这个实验日志。`
- `把这个想法转成下一轮任务。`

默认含义：Codex 自动读取 RD/NXT/EVO/研究记忆，按 T0-T7 分类，先做论文锚点检查，再调用对应角色，执行 Auto-Gate 保守裁决，写回研究债务、下一轮任务包、evolution report 和研究记忆。用户只在工具安装、算力投入、方向偏好、投稿、公开发布或强 claim 写入前确认。

当用户说“重新整理这批原始草稿”或类似请求时，默认先执行 T0 论文锚点归类，把草稿片段映射到论文主题、贡献候选和证据缺口；只有完成锚点分流后，才决定是否进入 T1/T2/T3 的公式整理、命题推进或模型整理。

角色系统必须服务当前 DRCC-MILP 主线：

```text
风模糊集 -> 决策相关能耗违约事件 -> DRCC reformulation -> 大规模巡检 MILP -> 论文 claim
```

- 日常科研推进默认由 `Auto-Gate 自动门禁裁决层` 保守自动裁决，负责系统内路由、验收、暂缓、降级、关闭候选、RD/NXT/EVO 和记忆写回。
- `PI导师` 保留为对外责任确认和重大方向确认角色，负责投稿、公开发布、最终强 claim 表述、重大主线取舍等人类责任节点，不要求用户亲自判断每个数学门禁。
- 副导师包括 `运筹最优化副导师`、`Wasserstein-DRO副导师`、`实验与Benchmark副导师`、`论文边界与新颖性副导师`，只有审查建议权。
- Codex 上层调度按任务类型 T0-T7 进行，不按 claim 编号调度；T0 负责论文方向与贡献策略，C-010/C-011 仅是当前优先 claim 示例。
- 每轮研究任务必须按 `08_agents工作流/自进化自动工作闭环协议.md` 收口：完成输入触发记录、论文锚点检查、T0-T7 分类、角色调用、产物验收、Auto-Gate 裁决、债务/记忆/角色变更写回、evolution report 和下一轮任务包。
- 普通整理和局部问答可使用轻量调用包；claim 状态变化、论文强 claim、角色变更、主线调整、关闭 `阻塞` 或 `claim阻断` 债务时，必须使用关键任务调用包。
- 执行专家和审计角色按 `角色职责矩阵.md` 交付产物，不得越权改变主线。
- 角色产物必须先按 `08_agents工作流/角色职责矩阵.md` 的“角色交付物验收口径”检查；验收后由 `研究债务协调员` 更新阶段小结，再进入 Auto-Gate 系统内裁决。
- 角色系统可随研究推进自进化，但必须经过“提出变更建议 -> 登记到角色变更记录 -> Auto-Gate 预审 -> 必要时人类责任确认 -> 更新角色矩阵/任务/claim”的闭环。
- 当前草稿、索引和 claim 表是状态快照，不是知识边界；缺口必须进入 `08_agents工作流/研究债务队列.md` 或标注为 `待查`、`待验证`、`文献边界未定`。
- 文献结论不得只停留在对话中；涉及 claim 边界、新颖性或 related work 的结论必须进入 `03_文献综述/文献边界表.md`、文献卡或研究债务队列。
- 研究缺口不得只留在对话中，必须进入 `08_agents工作流/研究债务队列.md`；角色结构变更不得只口头确认，必须进入 `08_agents工作流/角色变更记录.md`。
- `研究债务协调员` 负责登记、归档、催办和汇总研究债务，但不能绕过 Auto-Gate 关闭关键债务或改变 claim 状态，也不能自行关闭 `阻塞` 或 `claim阻断` 债务。
- 任何 claim 状态升级或写入论文强 claim 前，必须检查是否存在未关闭 `claim阻断` 债务；当前优先 claim 或任务推进后若无新增债务，也要登记“本轮无新增债务”。
- `claim阻断` 债务默认由 Auto-Gate 标记为 `继续阻断`；证据链齐备时只能进入 `关闭候选` 或 `系统内解除阻断候选`，对外强 claim 使用前仍需人类责任确认。
- 推前模糊集命题是辅助桥梁，不是 DRCC 优化主轴。
- SRI/CVaR/SOC 暂作为背景、对照或后续扩展，不进入当前主线 claim。

## 研究门禁

1. **假设账本门禁**：每个命题必须登记空间、拓扑、支撑集、紧性、可测性、矩条件、Wasserstein 阶数、连续性、对偶条件。
2. **反例优先门禁**：正式证明前必须检查边界例、退化例、空纤维、非紧支撑、不连续推前、闭包缺失、`W_1/W_2` 混用。
3. **证明审计门禁**：`proof-writer` 产出证明包后，由证明审计员给出 `通过 / 有条件通过 / 不通过`。
4. **运筹可解性门禁**：任何模型贡献必须说明是否凸、是否可对偶化、是否有限维可解、是否能进入巡检路径或调度优化。
5. **形式化门禁**：核心命题至少要有 Lean 风格陈述和形式化障碍说明；不要求第一版全量 Lean 证明。
6. **论文 claim 审批门禁**：论文中的强主张必须标注为 `定理已证 / 条件成立 / 数值支持 / 建模假设 / 文献已有 / 猜想`。

## 外部评审辅助通道

与网页版 GPT Pro 做外部评审对接时，可采用以下本地 agents 调度流程：

`08_agents工作流/Pro外部评审对接流程.md`

该流程是辅助审查通道，不属于论文工作调度主链，不替代 T0-T7、研究门禁、claim 审批、Auto-Gate 裁决或最终对外责任确认。Pro 反馈是外部审查建议，不关闭研究债务、不升级 claim；若影响 claim、角色、债务或任务包，必须写回研究债务队列、角色变更记录、evolution report 或下一轮任务包队列。

## 输出纪律

- 不得伪造文献、定理名、数值结果或证明结论。
- 若任务发现缺少解释器、包管理器、求解器、系统组件、外部许可证或其他工具，必须先向用户提出工具配置申请；未经用户明确确认，不得安装、升级或配置新工具，也不得把工具缺口只藏在 RD/NXT/EVO 或任务文本里。
- 证明不完整时必须标注 `条件成立`、`仅有推导`、`猜想` 或 `待验证`。
- 对 Wasserstein、DRO、DRCC 和 MILP 嵌入的论证必须检查阶数、地面距离、支撑集、矩条件、坏集闭性、半径平方、对偶条件和离散化精确性。
- 对可解性结论必须说明凸性、有限维化、强对偶或可计算替代表述。
- 所有项目产物必须遵守 `08_agents工作流/产物输出格式规范.md`；Markdown、LaTeX、PPT、图表、实验记录、审计报告和 claim 审批格式以该文件为准。
