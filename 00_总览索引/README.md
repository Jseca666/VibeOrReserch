# 项目地图

本工作台服务于风不确定性、无人机巡检、能耗风险度量与分布鲁棒优化相关的数学和运筹研究。

## 当前核心问题

1. 如何把原始风空间的 Wasserstein 模糊集作用到决策相关能耗违约事件上，得到 DRCC 所需的最坏违约概率或样本到坏集距离刻画。
2. 如何把风致能耗不确定性转化为巡检方案的分布鲁棒机会约束，并嵌入大规模巡检 MILP。
3. 如何保证模型贡献同时满足数学正确性、运筹可解性和论文 claim 可审计性。

## 第一案样板

第一条完整研究链路放在：

`02_定义命题证明/命题1_等价推前模糊集/`

该目录包含命题卡、假设账本、反例搜索记录、证明包、证明审计报告、Lean 风格陈述、运筹可解性说明和 claim 审批记录。

## 当前推导主线

乱序公式笔记的第一轮整理入口：

`00_总览索引/推导主线索引.md`

该索引把原始笔记重排为：

```text
风变量 -> 能耗映射 -> 决策相关违约集合 -> 样本到坏集距离 -> DRCC 有限维化 -> 大规模巡检 MILP -> 论文 claim
```

当前论文主线只关注分布鲁棒机会约束。推前模糊集命题是辅助桥梁，不是优化主轴；SRI/CVaR/SOC 暂作为背景、对照或后续扩展，不作为当前主线 claim。

使用规则：任何新命题进入 `02_定义命题证明/` 前，先在该索引中确认公式类型、来源位置、版本冲突和门禁风险。

## Markdown 公式规则

详细格式入口见：

`08_agents工作流/输出文档格式规范.md`

核心规则：行内公式使用 `\(...\)`，行间公式使用 `$$...$$`，不要用单美元 `$...$` 写行内数学公式；代码块和命令示例中的 `$` 除外。

## 每次推进研究时的最小流程

1. 在 `任务看板.md` 登记任务。
2. 在对应命题目录建立或更新命题卡。
3. 先补假设账本，再做反例搜索。
4. 使用 `proof-writer` 写证明包。
5. 使用证明审计员审查。
6. 若结果要进入论文，更新 `claim审批表.md`。
