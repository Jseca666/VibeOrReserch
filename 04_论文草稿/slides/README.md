# Slides

本目录存放成果型组会汇报材料，包括 beamer LaTeX、PPTX 和 markdown slides。

slides 产物必须进入 `08_agents工作流/产物产出队列.md` 或由 NXT/EVO 明确触发，且只承接已经通过文献边界、证明审计、反例测试或实验门禁的成果型内容。未成熟内容应留在 `technical_notes/`、proof package、RD/NXT/EVO 或实验日志中，默认暂缓生成 PPT。

## 子目录建议

- `beamer/`：数学汇报和公式密集型内部汇报。
- `pptx/`：导师组会、项目展示或非技术听众。
- `markdown/`：成果型汇报的快速草稿，不承载未验证想法。
- `assets/`：图、表、示意图和导出图片。

## 状态标签

每份汇报应标注页面或章节状态：

- `已证`
- `条件成立`
- `文献已有`
- `数值支持`

不得在成果型 PPT 主体中放入 `系统内候选`、`待验证`、`文献边界未定` 或 `claim阻断` 内容。若确需说明风险，只能作为“已确认限制”或“下一步已确定工作”，不得包装成成果。

公开发布或投稿相关汇报前，需要人类责任确认。
