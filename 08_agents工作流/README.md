# Agents 工作流

本目录定义本项目的 agent 路由、门禁协议和产物模板。

## Auto-Gate 常用入口

用户不需要先判断 T0-T7 或亲自承担数学 PI 终审。可以直接使用：

- `Auto-Gate 接管这轮输入。`
- `处理这份材料。`
- `审计这篇文献。`
- `检查这个实验日志。`
- `把这个想法转成下一轮任务。`

默认流程是：读取 RD/NXT/EVO/研究记忆 -> 自动分类 -> 调用角色 -> Auto-Gate 保守裁决 -> 写回队列、report 和研究记忆。用户只在工具安装、算力投入、方向偏好、投稿、公开发布或强 claim 写入前确认。

当用户说“重新整理这批原始草稿”时，默认先执行 T0 论文锚点归类，形成草稿片段到论文主题、贡献候选和证据缺口的映射，再决定是否进入 T1/T2/T3 的技术整理。

## 角色调用建议

- Codex 上层任务调度：先看 `Codex角色调用协议.md`，按 T0-T7 判断任务类型；T0 负责论文方向与贡献策略。
- 自动闭环运行：每轮任务按 `自进化自动工作闭环协议.md` 执行，完成论文锚点检查、T0-T7 分类、角色调用、产物验收、债务更新、记忆写回判断、evolution report 和下一轮任务包。
- Evolution report 归档：关键任务 report 归档到 `evolution_reports/`。
- 下一轮任务包追踪：生成的 NXT 任务包登记到 `下一轮任务包队列.md`。
- 角色职责总入口：先看 `角色职责矩阵.md`。
- 研究方向模糊：先由 Auto-Gate 生成候选任务、暂缓/降级建议或人类方向偏好确认项；副导师只给审查建议，不直接改变主线。
- DRCC-MILP 主线推进：优先调用 `DRCC reformulation 专家`、`大规模MILP算法专家`、`Wasserstein-DRO副导师`、`运筹最优化副导师`。
- 实验说服力：调用 `实验与Benchmark副导师` 和 `计算实验与Benchmark设计专家`。
- 文献边界：调用 `论文边界与新颖性副导师`；风不确定性、无人机巡检和能耗/功耗文献同步调用 `风-能耗物理建模专家`。
- T5 文献审查：文献分类、文献边界、新颖性和 related work 任务归入 `T5 文献边界与新颖性审查`，claim 编号只作为关联对象。
- 研究债务闭环：调用 `研究债务协调员`，维护 `研究债务队列.md` 和 `角色变更记录.md`。
- 角色产物验收：按 `角色职责矩阵.md` 中“角色交付物验收口径”检查后，再进入下一门禁。
- 产物输出格式：Markdown、LaTeX、PPT、图表、实验记录和审计报告先看 `产物输出格式规范.md`。
- 文献问题：先用 `paper-navigator`，再用 `research-survey`；结果必须进入文献分类总索引、文献边界表或研究债务队列。
- 公式推导：用 `formula-derivation`。
- 严格证明：用 `proof-writer`，然后必须用 `证明审计员` 审查。
- 论文规划：用 `paper-planning`。
- 论文自审：用 `paper-review`，但数学 claim 先看审批表。

## 标准流程

```text
问题/想法
  -> 建模对象卡
  -> DRCC 违约集合与命题卡
  -> 假设账本
  -> 反例搜索
  -> DRCC reformulation 证明包
  -> 证明审计与离散验证
  -> MILP 嵌入与可解性说明
  -> 实验与Benchmark设计
  -> 文献分类与边界审查
  -> 角色交付物验收
  -> 角色复盘与研究债务更新
  -> 研究债务协调员更新当前优先 claim/任务阶段小结
  -> 研究债务协调员汇总未关闭债务和角色变更建议
  -> claim阻断 债务检查
  -> Auto-Gate 系统内保守裁决
  -> evolution report 与下一轮任务包
  -> claim审批
  -> 论文草稿
```

## 角色自进化复盘

每轮当前优先 claim 或任务推进后必须检查：

- 当前角色职责是否仍匹配 DRCC-MILP 主线。
- 是否存在待查文献、待补假设、待做反例、待验证实验或待更新角色。
- 是否存在 `文献边界未定` 或未登记的文献分类、文献边界记录。
- 是否存在 `普通 / 阻塞 / claim阻断` 债务，尤其是会阻断 claim 状态升级的未关闭项。
- 是否需要提交 `角色职责矩阵.md` 中的“角色变更提案”。
- 是否需要由 `研究债务协调员` 把缺口登记到 `研究债务队列.md`。
- 若本轮无新增债务，也应记录“本轮无新增债务”。
- 角色结构变化必须写入 `角色变更记录.md`。
- 任何 claim 状态升级前，必须确认没有未关闭 `claim阻断` 债务。

## 外部评审辅助通道

需要让网页版 GPT Pro 辅助审查时，可按 `Pro外部评审对接流程.md` 准备本地临时核验包。该流程不是标准论文工作流主链，不替代 T0-T7 调度、claim 审批、研究债务关闭、角色变更落档、Auto-Gate 系统内裁决或最终对外责任确认。Pro 反馈影响项目状态时，必须经本项目门禁吸收并写回对应队列或 report。
