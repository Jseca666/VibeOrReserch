# 产物输出闭环协议

本文件规定研究过程产物如何升级为可追踪的正式输出产物。它补充 `产物输出格式规范.md`：格式规范回答“产物怎么写”，本协议回答“什么时候生成、进入哪里、状态如何标注、Auto-Gate 如何裁决、哪些用途被禁止”。

本协议不替代 RD/NXT/EVO、claim 审批、证明审计、文献边界审查或最终对外责任确认。未通过门禁的内容不得因为进入 LaTeX、slides、图表或表格而被视为已证结论。

## 1. 三层输出结构

本项目区分三层输出，避免把“已整理/部分验证的中间成果”误写成论文或汇报成果。

| 层级 | 承接内容 | 典型位置 | 边界 |
|---|---|---|---|
| 研究过程层 | RD、NXT、EVO、实验日志、原始推导、文献粗读 | `00_总览索引/`、`02_定义命题证明/`、`06_计算实验/`、`08_agents工作流/` | 记录过程，不作为正式输出 |
| 中间技术产物层 | 已验证推理片段、系统内候选命题、反例测试通过片段、文献支撑矩阵、待审计证明链 | `04_论文草稿/technical_notes/` | 可使用 LaTeX，但不属于论文正文或成果汇报 |
| 成果型输出层 | 已通过文献边界、证明审计、反例测试或实验门禁的论文片段和组会成果汇报 | `04_论文草稿/paper_progress/`、`04_论文草稿/main.tex`、`04_论文草稿/slides/` | 只承载成果型内容 |

默认路径为：研究过程层先生成证据；中间技术产物层承接可审计技术链；只有中间技术产物通过门禁并经 Auto-Gate 输出准入后，才能进入成果型输出层。

成果型输出还必须能追溯到 `04_论文草稿/paper_strategy/` 中的论文主题、贡献候选或贡献证据矩阵。若 technical note、proof package、paper progress 或 slides 无法说明服务的论文锚点，Auto-Gate 默认暂缓成果型输出，只保留为内部技术债务或中间技术产物。

## 2. 产物类型

| 类型 | Output 前缀 | 层级 | 用途 | 默认位置 |
|---|---|---|---|---|
| technical note | `OUT-TN` | 中间技术产物 | 内部技术证明链和推导说明 | `04_论文草稿/technical_notes/` |
| derivation log | `OUT-DL` | 中间技术产物 | 逐步数学推导日志，记录每一步的类型、假设、文献支撑和待审计边界 | `04_论文草稿/technical_notes/derivation_logs/` |
| counterexample note | `OUT-CE` | 中间技术产物 | 反例、失败机制或边界击穿说明 | `04_论文草稿/technical_notes/counterexample_notes/` |
| validation record | `OUT-VR` | 中间技术产物 | 串联推导、文献、脚本结果、EVO 和未关闭 RD 的验证索引 | `04_论文草稿/technical_notes/validation_records/` |
| proof package | `OUT-PP` | 中间技术产物 | 供证明审计员复核的正式证明包 | `04_论文草稿/technical_notes/proof_packages/` |
| theorem card | `OUT-TC` | 中间技术产物 | 单个命题的 LaTeX 状态卡 | `04_论文草稿/technical_notes/theorem_cards/` |
| appendix candidate | `OUT-APP` | 中间技术产物 | 候选附录材料，尚未进入正文 | `04_论文草稿/technical_notes/appendix_candidates/` |
| paper progress | `OUT-PG` | 成果型输出 | 成果型论文 LaTeX 进度片段 | `04_论文草稿/paper_progress/` |
| slides / beamer / PPT | `OUT-SL` | 成果型输出 | 成果型组会汇报材料 | `04_论文草稿/slides/` |
| figure / table | `OUT-FIG` / `OUT-TAB` | 视来源而定 | 图、表、实验结果展示 | `04_论文草稿/figures/` 或 `04_论文草稿/tables/` |
| literature support matrix | `OUT-LIT` | 研究过程或中间技术产物 | 文献支撑和边界矩阵 | `03_文献综述/` 或 technical note 附表 |

## 3. 触发条件

出现以下任一情况时，Auto-Gate 必须判断是否生成 OUT 产物：

- NXT 完成后产生可追踪技术链、模型链、文献边界或实验日志。
- EVO report 裁决为 `自动有条件通过`、`系统内解除阻断候选` 或要求沉淀证明链。
- 命题链达到 `系统内条件成立候选`，但仍不允许进入论文强 claim。
- 工作链中存在关键推导步骤、反例击穿机制或脚本验证结果，需要从状态摘要升级为可审计推导日志、反例笔记或验证记录。
- 文献边界、证明审计或实验验证推进到需要论文进度片段。
- 用户需要阶段汇报、组会汇报或对外展示草稿。
- 图表、实验表格、文献矩阵或证明附录可能进入论文或 slides。

若内容仍属于猜想、未审查 claim、一次性异常或未关闭 `claim阻断` 债务，Auto-Gate 默认只允许生成研究过程记录或中间技术产物，不允许进入成果型论文 LaTeX 或成果型组会 PPT。

## 4. 状态字段

产物状态使用：

- `待生成`
- `草案`
- `内部技术说明`
- `系统内候选`
- `待证明审计`
- `待文献边界`
- `被 RD 阻断`
- `论文草稿候选`
- `可内部汇报`
- `禁止强 claim`
- `暂缓`
- `作废`

每个 OUT 产物必须写明：

| 字段 | 内容 |
|---|---|
| Output ID | `OUT-YYYYMMDD-序号` |
| 来源 | NXT / EVO / RD / 命题卡 / 文献边界 / 实验日志 |
| 产物类型 | technical note / derivation log / counterexample note / validation record / proof package / theorem card / paper progress / slides / figure / table / literature matrix |
| 产物层级 | 研究过程 / 中间技术产物 / 成果型输出 |
| 目标路径 | 文件路径 |
| 关联 claim | 无 / claim 编号 |
| 当前状态 | 状态字段 |
| Auto-Gate 裁决 | 生成 / 暂缓 / 只进入 RD/NXT / 只写 internal note / 可进 paper progress / 可进 slides / 不得进入 main.tex |
| 禁止用途 | 不得写原创强 claim / 不得作为正式 benchmark / 不得公开发布 / 不得进入定理环境 |
| 人类确认需求 | 无 / 工具安装 / 投稿前 / 公开发布前 / 强 claim 写入前 |

## 5. Auto-Gate 输出裁决

Auto-Gate 对输出产物采用保守默认：

- 证据不足时，生成 RD/NXT，不生成成果型论文或 slides 产物。
- 命题链达到系统内候选但仍有文献或证明阻断时，只生成 technical note、theorem card 或 proof package。
- 若中间技术产物只记录状态而缺少可审计推导过程，Auto-Gate 应优先生成 derivation log、counterexample note 或 validation record，再判断是否需要 proof package。
- 文献边界显示已有结果时，自动降级为背景、结构化应用或引用说明，不写原创强 claim。
- 小规模实验通过时，只生成实验日志、技术表格或中间技术说明，不写正式 benchmark。
- 未经证明审计、文献边界审查和 Auto-Gate 输出准入，不得进入 `paper_progress/`、`main.tex` 或 `slides/`。
- 公开 slides、投稿正文、最终强 claim 写入前必须有人类责任确认。

Auto-Gate 输出裁决路径使用：

- `生成中间技术产物`
- `保持中间层`
- `升级为成果型输出候选`
- `暂缓成果型输出`
- `禁止进入成果型输出`

`系统内候选` 默认只能进入中间技术产物层；只有通过对应门禁后，才能升级为成果型输出候选。

## 6. RD/NXT/EVO 联动

- NXT 队列负责研究动作；OUT 队列负责外显产物生命周期。两者互相引用但不能互相替代。
- NXT 完成后若触发正式产物，必须在 `输出产物触发` 字段写入 OUT-ID。
- EVO report 必须记录本轮是否触发 OUT 产物、产物状态和禁止用途。
- RD 未关闭时，OUT 产物必须标注对应 RD 阻断，不能通过文件格式把未审查内容包装成已完成结论。
- OUT-PG 和 OUT-SL 必须引用已通过门禁的中间技术产物或成果证据作为来源。
- OUT-PG、OUT-SL、OUT-PP 和重要 OUT-TN 必须引用 paper strategy 或 contribution evidence matrix，说明服务的论文主题、贡献候选和证据缺口。

## 7. C-010/W1 当前输出边界

截至 EVO-20260502-016，C-010 的 \(W_1\) DRCC 工作链属于中间技术产物层，只允许进入：

- internal technical note；
- theorem card；
- derivation log；
- counterexample note；
- validation record；
- 后续 proof package 候选。

禁止进入：

- `main.tex` 的原创强定理；
- 投稿论文强贡献；
- 对外 slides 的确定性强 claim；
- `claim阻断` 债务关闭依据。

可写边界为：通用 Wasserstein DRCC reformulation 属于已有强相关文献；本项目只能将其作为风-能耗映射 \(G(z)\)、二维 \(z\) 网格、航向能耗相关性和巡检 MILP 结构化适配的技术基础候选。
