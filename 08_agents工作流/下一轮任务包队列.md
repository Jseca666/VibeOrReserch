# 下一轮任务包队列

本文件记录 evolution report 生成的下一轮任务包，用于跨轮次追踪后续动作。它不替代 `研究债务队列.md`、claim 审批表、角色变更记录或 `PI导师` 裁决。

## 1. 状态约定

- `待启动`
- `进行中`
- `已完成`
- `暂缓`
- `作废`

## 2. 任务包队列表

| 任务包 ID | 来源 | 建议任务类型 | 关联 claim | 目标产物 | 必须调用角色 | 阻断条件 | PI裁决条件 | 状态 | 对应 report |
|---|---|---|---|---|---|---|---|---|---|
| NXT-YYYYMMDD-001 | 债务 / PI / T7 / 文献 / 实验 / 反例 / 论文草稿 / 记忆写回 | T1-T7 | 无 / claim 编号 / 待定 | 目标产物 | 角色列表 | 无 / RD-ID / 文献边界未定 / 实验设计未定 | 无 / 待PI裁决 | 待启动 / 进行中 / 已完成 / 暂缓 / 作废 | EVO-YYYYMMDD-序号 |

## 3. 记录规则

- 每个 `NXT-YYYYMMDD-序号` 必须来自一个 evolution report、研究债务、角色变更建议、PI裁决事项或阶段小结。
- 由 `阻塞` 或 `claim阻断` 债务生成的任务包，必须在 `阻断条件` 中写入对应 RD 编号。
- 涉及 claim 状态变化、论文强 claim、主线调整或角色结构变化的任务包，必须在 `PI裁决条件` 中写明 `待PI裁决`。
- `研究债务协调员` 可以登记和汇总任务包，但不能自行关闭关键债务、升级 claim 或改变角色结构。
- 若任务包暂缓，必须说明暂缓原因、重新触发条件，并同步更新对应 report 或研究债务记录。

## 4. 边界

- 下一轮任务包队列只追踪下一步动作，不证明命题、不审查文献、不验证实验。
- 任务包状态 `已完成` 只表示该任务动作已完成，不表示 claim 已升级、债务已关闭或角色变更已生效。
- C-010/C-011 只是当前优先 claim 示例；任务包队列不按 claim 编号作为固定入口。
