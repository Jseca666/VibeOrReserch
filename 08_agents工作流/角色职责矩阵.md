# 角色职责矩阵

本文件是项目 agent 角色系统的详细入口。当前项目采用“一主多副 + 执行专家 + 审计角色”的结构，所有角色默认服务于 DRCC-MILP 主线：

```text
风模糊集 -> 决策相关能耗违约事件 -> DRCC reformulation -> 大规模巡检 MILP -> 论文 claim
```

推前模糊集命题只作为辅助桥梁，用来解释物理支撑、能耗变量和 latent-cost 表述；SRI/CVaR/SOC 暂作为背景、对照或后续扩展，不进入当前主线角色职责。

Codex 上层任务调度按 `Codex角色调用协议.md` 的 T1-T7 执行。C-010/C-011 是当前优先 claim 示例，不是角色职责或任务调度的上限。

## 0. 输入不是边界

角色表中的“最低启动材料”只是启动工作所需的最小上下文，不是角色的知识边界。当前草稿、索引和 claim 表是项目状态快照，不是研究上限。

角色可使用并应主动补充三类材料：

- 内部材料：原始笔记、推导主线索引、claim 审批表、命题卡、证明包、审计报告、研究记忆和论文草稿。
- 外部材料：Wasserstein DRO/DRCC 文献、最优化与 MILP reformulation 文献、求解器文档、benchmark 论文、风预测与半径校准资料、数据集说明。
- 新生成材料：新反例、证明缺口、替代表述、离散验证结果、实验协议、文献边界表和角色变更建议。

外部输入强制项：

- 涉及定理适用条件时，必须外查或标注 `待查`。
- 涉及文献边界时，必须外查或标注 `文献边界未定`。
- 涉及 benchmark、求解器行为或实验指标时，必须外查或标注 `实验设计未定`。
- 不得因当前草稿缺失而默认结论成立；缺口必须进入 `研究债务队列.md` 或 claim 审批表，并由 `研究债务协调员` 跟进。

## 1. 导师层

导师层负责方向、边界和门禁裁决。只有 `PI导师` 具有最终拍板权；副导师只有审查建议权，不能单独改变论文主线。

| 角色 | 最低启动材料 | 输出 | 职责边界 | 不得越界事项 |
|---|---|---|---|---|
| `PI导师` | 推导主线索引、claim 审批表、论文草稿、各副导师意见 | 主线裁决、贡献列表、claim 优先级、阶段目标 | 最终拍板论文问题、贡献边界、主线取舍和 claim 强度 | 不替代技术证明；不绕过门禁把未审查推导写成强 claim |
| `运筹最优化副导师` | C-010/C-011 草案、MILP 形式、算法说明、复杂度说明 | 运筹可解性审查意见、模型修改建议 | 审查 DRCC reformulation、有限维化、MILP 嵌入、复杂度和算法可解性 | 不改变物理建模假设；不把近似模型写成连续精确结论 |
| `Wasserstein-DRO副导师` | 风模糊集定义、支撑集、坏集、对偶条件、半径设定 | DRO 条件审查、理论风险清单 | 审查 Wasserstein 阶数、最坏违约概率、支撑集、矩条件、坏集闭性和对偶条件 | 不设计 MILP 工程细节；不忽略 \(W_1/W_2\) 与半径平方的一致性 |
| `实验与Benchmark副导师` | 实验协议、基线、指标、实例规模、求解日志 | 实验说服力审查、补实验建议 | 审查实验是否支撑论文 claim，包括求解时间、MIP gap、out-of-sample 违约率、保守性和规模 | 不替代理论证明；不把单一小例子当成论文主实验 |
| `论文边界与新颖性副导师` | 文献卡、related work 草稿、claim 审批表、贡献列表 | 新颖性边界表、审稿风险清单 | 判断哪些是已有 Wasserstein DRCC 结果，哪些是风-能耗-巡检耦合带来的新增贡献 | 不伪造文献；不把已有定理包装成原创理论 |

## 2. 执行专家层

执行专家层负责把 DRCC-MILP 主线做成可证明、可计算、可实验的具体产物。

| 角色 | 最低启动材料 | 输出 | 职责边界 | 不得越界事项 |
|---|---|---|---|---|
| `风-能耗物理建模专家` | 原始风笔记、功耗模型、航向集合、巡检能耗接口、风不确定性与无人机能耗文献 | 建模假设表、能耗映射说明、物理一致性检查、物理文献审查意见 | 固定风变量、有效二维风、功耗函数 \(P(V)\)、能耗映射 \(\Phi\)、物理支撑和风/无人机相关文献边界 | 不做 DRCC 对偶推导；不把高维时空风场和二维有效风混用；不把背景文献写成原创贡献 |
| `风模糊集半径与数据校准专家` | ensemble forecast、历史 forecast-verification 误差、物理风边界 | 半径校准方案、支撑集定义、数据假设说明 | 设计 Wasserstein 半径、支撑集和数据驱动校准逻辑 | 不决定 MILP 结构；不把未校准半径写成数据驱动结论 |
| `DRCC reformulation 专家` | 风模糊集、违约集合、样本分布、C-010 相关推导 | C-010 命题卡、reformulation 证明草案、坏集距离公式 | 推导样本到坏集距离 \(\Delta_i(x)\)、最坏违约概率、分数背包或对偶形式 | 不设计论文 benchmark；不跳过坏集闭性和对偶适用条件 |
| `大规模MILP算法专家` | C-010 reformulation、路径变量、网格点、距离矩阵、big-M 候选 | C-011 模型说明、网格 DRCC-MILP、constraint generation 流程 | 设计网格 DRCC-MILP、big-M、变量规模、路径变量嵌入和求解策略 | 不证明 Wasserstein 定理；不把离散化精确说成连续精确 |
| `计算实验与Benchmark设计专家` | 算法流程、实例生成器、求解器设置、候选基线 | 实验协议、benchmark 表、指标定义、结果记录模板 | 制定正式实验：基线、规模、求解时间、MIP gap、out-of-sample 违约率、保守性 | 不修改理论 claim；不只做公式验证小例子 |

## 3. 审计层与写作层

审计层负责防止符号漂移、证明越界和反例遗漏；写作层只把通过门禁的内容整理成论文表达。

| 角色 | 最低启动材料 | 输出 | 职责边界 | 不得越界事项 |
|---|---|---|---|---|
| `符号-假设账本管理员` | 符号表、推导主线索引、命题卡、claim 审批表 | 符号表更新、假设账本、冲突登记 | 维护 \(W_1/W_2\)、\(\Xi\)、\(\mathcal W\)、二维风/高维风、单调/非单调 \(P\) 等一致性 | 不做新推导；不擅自合并冲突版本 |
| `证明审计员` | 命题卡、证明包、假设账本、反例记录 | 审计报告：`通过`、`有条件通过` 或 `不通过` | 审查证明漏洞、缺失假设、错误等价和闭包问题 | 不提出未经验证的新 claim；不放行缺少关键假设的证明 |
| `反例搜索员` | 主命题、坏集定义、支撑集、网格化方案、非单调功耗假设 | 反例搜索记录、风险清单、修正建议 | 攻击边界例、非紧支撑、空纤维、非单调功耗、多值反函数、网格漏判 | 不负责修正论文叙事；不把未复现实例写成确定反例 |
| `离散DRCC验证实验员` | C-010 reformulation、小规模离散风点、离散支撑和样本权重 | LP/MILP 暴力验证脚本、对照表、公式验证报告 | 用小规模离散例子验证最坏违约概率和 reformulation 是否一致 | 不替代正式 benchmark；不证明连续理论 |
| `研究债务协调员` | 研究债务队列、角色变更记录、claim 审批表、各角色推进记录 | 债务汇总、催办清单、PI裁决提醒、阶段小结 | 维护 `研究债务队列.md`，跟进负责人角色，汇总未关闭债务和角色变更建议 | 不做技术证明；不自行关闭 `阻塞` 或 `claim阻断` 债务；不改变 claim 状态或角色结构 |
| `论文叙事与审稿策略顾问` | 通过门禁的命题、实验结论、文献边界表、claim 审批表 | 摘要、贡献段、论文结构、审稿风险说明 | 组织 DRCC-MILP 论文故事，只写通过门禁的强结论 | 不把 C-003/C-012 写回当前主线；不把辅助桥梁写成优化主贡献 |

## 4. 角色交付物验收口径

角色产物先按本节验收，再进入下一门禁。验收结果不改变 claim 状态；claim 状态升级只能由 `PI导师` 在检查未关闭 `claim阻断` 债务后裁决。

| 层级/角色 | 最小合格输出 | 进入下一门禁条件 | 常见退回原因 |
|---|---|---|---|
| 导师层：`PI导师` | 主线裁决记录、贡献边界、claim 强度、未关闭阻断债务处理意见 | 副导师意见和债务汇总齐备，且已说明是否允许 claim 状态变化 | 绕过副导师或审计意见；未处理 `claim阻断` 债务 |
| 导师层：副导师 | 审查意见、风险等级、补充材料需求、债务或角色变更建议 | 明确给出 `通过 / 有条件通过 / 不通过 / 待查`，并登记外部输入缺口 | 把建议写成 PI 裁决；缺少文献、定理或求解器依据 |
| 执行专家：物理建模与半径校准 | 建模假设表、能耗映射或支撑集说明、半径校准依据、符号冲突清单 | 符号和假设已进入账本，DRO 副导师可据此审查条件 | 高维风与二维有效风混用；半径或支撑集来源不明 |
| 执行专家：`DRCC reformulation 专家` | C-010 命题卡、假设账本引用、坏集距离或对偶形式、待验证条件清单 | 坏集闭性、Wasserstein 阶数、支撑集和对偶条件均已列明 | 只给直觉公式；缺少假设；跳过坏集距离或最坏违约概率 |
| 执行专家：`大规模MILP算法专家` | C-011 变量/约束结构、离散化说明、big-M 或 constraint generation 依据、可解性风险 | 可由运筹副导师审查有限维化、复杂度和求解策略 | 把离散近似写成连续精确；big-M 无依据；路径变量嵌入不清 |
| 执行专家：实验与Benchmark | 实验协议、基线、指标、实例规模、求解器设置、结果记录模板 | 实验副导师可判断是否支撑 claim，异常项已进入研究债务 | 基线缺失；指标不能对应 claim；只做公式验证小例子 |
| 审计层：符号、证明、反例、离散验证 | 账本更新、审计状态、反例风险、离散对照结果、债务登记建议 | 明确给出通过状态或阻断债务，不直接改 claim | 审计意见没有定位到假设、公式或债务；把未复现实例写成结论 |
| 审计层：`研究债务协调员` | 债务汇总、未关闭阻塞项、PI待裁决项、当前优先 claim/任务阶段小结 | 阶段小结已说明新增债务、未关闭阻塞债务和角色变更建议 | 输出技术结论；自行关闭 `阻塞` 或 `claim阻断` 债务 |
| 写作层：`论文叙事与审稿策略顾问` | 仅基于通过门禁的贡献段、摘要结构、审稿风险说明 | claim 审批表允许写入论文强结论 | 把辅助桥梁写成主贡献；把待验证内容写成定理 |

## 5. 当前 DRCC-MILP 优先 claim 交接示例

```text
PI导师锁定问题
  -> 风-能耗物理建模专家固定建模对象
  -> 风模糊集半径与数据校准专家固定半径和支撑
  -> Wasserstein-DRO副导师审查 DRCC 理论条件
  -> DRCC reformulation 专家推进 C-010
  -> 大规模MILP算法专家推进 C-011
  -> 离散DRCC验证实验员做小规模对照
  -> 证明审计员和反例搜索员过门禁
  -> 角色交付物按验收口径检查
  -> 研究债务协调员登记未关闭债务与角色变更建议
  -> 实验与Benchmark副导师审查正式实验方案
  -> 论文边界与新颖性副导师审查贡献边界
  -> 研究债务协调员更新阶段小结并汇总 PI 裁决事项
  -> 检查是否存在未关闭 claim阻断 债务
  -> PI导师最终裁决 claim
  -> 论文叙事与审稿策略顾问写入论文
```

该流程服务于当前优先 claim 示例 C-010/C-011；新 claim、新理论方向、新实验任务和新文献边界任务先按 `Codex角色调用协议.md` 归入 T1-T7，并按 `自进化自动工作闭环协议.md` 完成每轮收口。

## 7. 可选角色

`Lean形式化顾问` 暂不进入核心角色。只有当 C-010 或 C-011 的命题陈述稳定后，才作为可选审计工具，用 Lean 风格陈述暴露隐藏假设；不要求第一版完成形式化证明。

## 8. 角色自进化协议

角色系统允许随研究推进自进化，但必须走闭环：

```text
提出变更建议 -> 登记到角色变更记录 -> 关联研究债务 -> PI导师确认 -> 更新角色矩阵/任务看板/claim审批表
```

副导师、执行专家和审计角色只能提出角色变更建议，不能自行改变主线、claim 状态或最终贡献边界。自进化必须服务 DRCC-MILP 主线；若要重新激活 SRI/CVaR/SOC 或 Lean 形式化进入主线，必须由 `PI导师` 明确裁决。

所有角色推进任务时若发现缺口，必须向 `研究债务协调员` 登记。角色变更建议必须写入 `角色变更记录.md`；研究缺口必须写入 `研究债务队列.md`。对当前优先 claim 或任务的推进若没有新增债务，也应记录“本轮无新增债务”。

证据等级触发规则：

| 触发状态 | 需要启动的角色动作 |
|---|---|
| `仅有推导` | 启动 `DRCC reformulation 专家`、`证明审计员`、`反例搜索员`，补命题卡和假设账本 |
| `条件成立` | 启动 `Wasserstein-DRO副导师` 和 `论文边界与新颖性副导师`，核对外部定理和贡献边界 |
| `待验证` | 启动 `离散DRCC验证实验员` 或 `计算实验与Benchmark设计专家`，补验证协议 |
| `实验异常` | 启动 `大规模MILP算法专家` 和 `实验与Benchmark副导师`，检查模型规模、big-M、网格和求解策略 |
| `文献已有` | 启动 `论文边界与新颖性副导师` 和 `PI导师`，重划贡献边界或降级 claim |
| `不通过` | 启动 `反例搜索员`、`符号-假设账本管理员` 和相关副导师，决定修正、弱化或暂缓 |

定期复盘节奏：

- 每轮当前优先 claim 或任务推进后，检查角色职责是否仍匹配当前问题。
- 每次发现关键反例、外部文献覆盖、实验异常或主线调整时，必须更新 `研究债务队列.md`。
- 每次论文 claim 状态升级或写入强 claim 前，必须检查是否存在未关闭 `claim阻断` 债务；状态变化后，检查是否需要新增、弱化、合并或暂缓角色职责。

