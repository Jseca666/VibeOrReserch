## 2. 层级约定

- `研究过程`：RD、NXT、EVO、实验日志、原始推导和文献粗读。
- `中间技术产物`：technical note、derivation log、counterexample note、validation record、theorem card、proof package、appendix candidate，用于承接已验证推理片段、系统内候选命题、反例测试通过片段、文献支撑矩阵、待审计证明链和反例推理机制。
- `成果型输出`：paper progress、`main.tex`、slides，只承接已通过文献边界、证明审计、反例测试或实验门禁的成果型内容。

## 4. 记录规则

- 每个 OUT 必须能追溯到 NXT/EVO/RD 或具体产物。
- OUT 状态 `草案` 或 `系统内候选` 不代表 claim 已升级。
- OUT 状态 `内部技术说明` 不代表可以进入论文正文。
- 中间技术产物进入 `paper_progress/`、`main.tex` 或 `slides/` 前，必须经过 Auto-Gate 成果型输出准入。
- OUT-PG 或 OUT-SL 必须引用已通过门禁的中间技术产物或成果证据作为来源。
- 若需要编译 LaTeX、导出 PDF、生成 PPTX 或安装工具，必须显式申请工具配置。

## 5. 当前边界

C-010/W1 当前只能作为中间技术产物生成 internal technical note 与 theorem card。proof package、appendix candidate、paper progress 和 slides 均需后续 Auto-Gate 裁决，不能由本队列自动升级。

本轮新增的 derivation log、counterexample note 和 validation record 只补足中间验证过程，不改变上述成果型输出边界。
