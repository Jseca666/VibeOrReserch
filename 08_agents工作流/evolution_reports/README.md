# Evolution Reports 归档说明

本目录用于保存每轮自进化自动工作闭环结束后的 evolution report。它是跨轮次复盘证据，不是 claim 审批表、研究债务队列或角色变更记录的替代品。

自 NXT-20260502-008 起，历史 report 中的 `PI导师`、`PI待裁决事项` 等字样按 Auto-Gate 系统内裁决与人类确认边界解释；用户无需承担数学终审。

## 1. 命名规则

- 文件名使用 `EVO-YYYYMMDD-序号.md`。
- `轮次 ID` 必须与文件名一致。
- 同一天多轮闭环按 `001`、`002`、`003` 递增。

## 2. 必须归档的情况

以下任务不得只在对话中给出 report，必须归档：

- 使用关键任务调用包。
- 新增、更新、关闭或暂缓研究债务。
- 提出 claim 状态变化请求或论文强 claim 写入请求。
- 出现角色变更建议、T7 判断或角色职责冲突。
- 出现 Auto-Gate 裁决事项或人类确认需求。
- 生成下一轮任务包。
- 触发研究记忆写回建议。

轻量问答若不产生 report，必须在当轮回复或相关阶段小结中说明“本轮为轻量问答，不产生 report；无新增债务；无下一轮任务包”。

## 3. 边界

- evolution report 只记录本轮闭环状态。
- evolution report 不替代 claim 审批表。
- evolution report 不关闭研究债务。
- evolution report 不改变角色结构。
- evolution report 不替代 Auto-Gate 门禁裁决，也不替代最终对外责任确认。
- report 中出现的人类确认需求，必须在工具安装、算力投入、投稿、公开发布、最终强 claim 表述或重大方向取舍前获得确认。

## 4. 轻量 EvoScientist 对齐

当前只做轻量对齐，不建立完整 EvoScientist 独立映射协议。

| EvoScientist 风格层 | 本项目对应位置 | 作用 |
|---|---|---|
| IDE | 输入触发源、T1-T7 分类、下一轮任务包 | 发现和生成研究候选任务 |
| IVE | 角色交付物验收、证明审计、实验验证、研究债务队列、Auto-Gate 裁决与人类确认事项 | 验证、筛选和阻断不合格结论 |
| ESE | 研究记忆写回、T7 触发、角色变更记录、evolution report | 沉淀经验并推动系统自进化 |

## 5. 最小模板引用

report 正文模板见 `../自进化自动工作闭环协议.md` 的“Evolution Report 模板”。如本轮生成下一轮任务包，必须同步登记到 `../下一轮任务包队列.md`。
