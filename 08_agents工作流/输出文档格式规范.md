# 输出文档格式规范

本文件是项目 Markdown 产物的子规范。所有产物的格式总入口为 `08_agents工作流/产物输出格式规范.md`；后续整理笔记、命题卡、证明包、审计报告、研究记忆和论文草稿说明时，Markdown 部分默认遵守本规范。

## 1. 基本语言与文件

- 默认使用中文写作、中文标题和中文目录。
- Lean、Sage、LaTeX、Python 等工具内部文件可使用英文名，保证工具兼容。
- Markdown 文档使用 `.md` 后缀，避免把数学公式放进反引号代码片段中。

## 2. 数学公式

当前 Markdown 渲染环境采用以下公式写法：

- 行内公式使用 `\(...\)`。
- 行间展示公式使用 `$$...$$`。
- 不使用单美元形式写行内数学公式；当前环境会把它显示成 LaTeX 代码或造成渲染不稳定。
- 代码块、反引号代码片段和命令示例中的美元符号不按数学公式处理。例如 PowerShell 环境变量写法是命令文本，不是数学公式。

示例：

```markdown
行内：令 \(W_2(P,\widehat Q)\le \varepsilon\)。

行间：
$$
\Phi_\# P = R.
$$
```

## 3. 公式整理标注

整理数学推导时，每个非平凡公式应尽量标注类型：

- `定义`
- `恒等式`
- `条件命题`
- `近似`
- `解释`
- `优化改革写`
- `待验证`

证明不完整或假设未锁定时，不能把公式写成无条件定理。

## 4. 表格与状态

涉及 claim、命题、证明或实验状态时，优先用表格记录：

| 项 | 建议字段 |
|---|---|
| 命题 | 编号、陈述、依赖、状态、主要缺口 |
| claim | 编号、论文表述、证据状态、审批状态、备注 |
| 推导链 | 顺序、公式/对象、类型、来源、风险 |
| 反例搜索 | 风险点、构造、结果、处理建议 |

状态词应使用项目约定，例如 `定理已证`、`条件成立`、`仅有推导`、`待验证`、`猜想`、`不通过`。

## 5. 检查规则

交付 Markdown 文档前至少检查：

- 普通正文中没有裸单美元数学公式。
- `$$` 行间公式分隔符成对出现。
- 代码块和命令示例没有被误改成数学公式。
- 强 claim 已在 `00_总览索引/claim审批表.md` 或对应命题目录登记。
- 公式类型、来源笔记和关键假设没有缺失。
