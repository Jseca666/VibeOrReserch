# 产物输出格式规范

本文件是项目所有研究产物的格式总入口。Markdown、LaTeX、PPT/slides、图表、实验记录、审计报告、claim 审批和文件命名都以本文件为准。

旧文件 `输出文档格式规范.md` 保留为 Markdown 子规范；涉及 Markdown 细节时优先查看该文件。

## 1. 通用原则

- 默认使用中文写作、中文标题和中文目录。
- Lean、Sage、LaTeX、Python、BibTeX、脚本和工程文件可使用英文名，保证工具兼容。
- 不伪造文献、定理名、证明结论、实验结果、求解日志或数值指标。
- 未通过门禁的内容必须标注为 `待验证`、`仅有推导`、`条件成立`、`猜想`、`文献边界未定` 或 `实验设计未定`。
- 强 claim 必须能追溯到 claim 审批表、命题卡、证明审计、实验记录或研究债务队列。

## 2. Markdown 文档

- Markdown 细节遵守 `08_agents工作流/输出文档格式规范.md`。
- 行内公式使用 `\(...\)`。
- 行间公式使用 `$$...$$`。
- 不使用单美元形式写行内数学公式。
- 表格用于记录 claim、命题、推导链、研究债务、角色变更、实验状态和审计状态。
- 公式推导文档应区分 `定义`、`恒等式`、`条件命题`、`近似`、`解释`、`优化改革写` 和 `待验证`。

## 3. 文献综述与 BibTeX

- 文献工作应使用文献卡、文献分类总索引和文献边界表管理，不只停留在对话中。
- 文献卡至少记录类别、研究功能、阅读状态、关联任务类型、关联 claim、可支持/不能支持的结论和 `文献已有` 风险。
- 风不确定性和无人机相关文献应记录风模型、任务场景、能耗假设、路径/调度模型和实验数据来源。
- 文献边界表应区分外部已有结论与本项目新增贡献；影响强 claim 且边界未定时应进入研究债务队列。
- BibTeX 条目必须来自真实文献来源，未查实文献不得进入正式参考文献库。

## 4. LaTeX 产物

- 论文源码、补充材料、定理环境、公式编号、引用和 BibTeX 统一归入论文或 LaTeX 目录。
- 定理、引理、命题、推论和假设应使用一致的编号与交叉引用。
- 论文中的强 claim 必须对应已登记的 claim 状态，不得把 `仅有推导` 或 `待验证` 写成定理。
- 参考文献必须有真实来源；未查实文献不得进入 BibTeX。
- 编译日志、错误修复记录和最终 PDF 输出应可追溯。

## 5. PPT 与 Slides

- 汇报材料应围绕问题、挑战、方法、证据、结论组织，不把未通过门禁的内容写成确定结论。
- 每页标题应表达该页结论或目的，避免只写宽泛主题。
- 公式页优先展示关键对象、假设和结论，避免堆叠长推导。
- 论文 claim、实验结果和图表必须能追溯到对应文档或数据来源。
- 导出文件应保留源文件和可展示版本，例如 PPTX、PDF 或图片导出版本。

## 6. 图表与流程图

- 图表必须包含图题、编号或文件名、数据来源、生成脚本或生成说明。
- Mermaid 图、流程图和论文图应保存源文件，必要时同时导出 PNG、SVG 或 PDF。
- 实验图必须能追溯到原始数据、处理脚本和绘图参数。
- 概念图不得伪装成实验结果；示意图和真实结果图必须区分。

## 7. 实验记录与结果表

- 实验记录应包含任务目标、关联 claim、数据来源、实例规模、随机种子、求解器或运行环境、指标和结论边界。
- MILP、LP、DRO 或仿真实验必须记录求解状态、运行时间、gap、可行性、异常和失败原因。
- out-of-sample 验证、保守性、违约率、运行时间和规模等指标应与论文 claim 对齐。
- 实验异常、不可复现结果、基线缺失和求解器行为不明时，应进入研究债务队列。

## 8. 审计报告与 Claim 审批

- 证明审计报告至少给出 `通过`、`有条件通过` 或 `不通过`，并说明关键假设、漏洞和阻断项。
- claim 审批记录应标注 `定理已证`、`条件成立`、`数值支持`、`建模假设`、`文献已有` 或 `猜想`。
- 存在未关闭 `claim阻断` 债务时，不得升级论文强 claim。
- 研究债务阶段小结应说明新增债务、未关闭阻塞项、PI待裁决事项和下一步。
- evolution report 和下一轮任务包属于项目闭环产物，必须包含 ID、来源、状态、关联债务或 claim，并能追溯到 `evolution_reports/`、`下一轮任务包队列.md` 或研究债务队列。
- evolution report 不替代 claim 审批表、研究债务关闭、角色变更记录或 `PI导师` 裁决。

## 9. 文件命名与归档

- 文件名应表达产物类型和研究对象，避免只用 `new`、`draft`、`final` 等模糊名称。
- 原始笔记不直接覆盖；整理、证明、审计和论文产物应进入对应目录。
- 重要产物应能从 `00_总览索引/`、claim 审批表或角色工作流文件追溯。
- 阶段性产物可以保留日期或轮次 ID，便于复盘和研究记忆写回。

## 10. 检查清单

- 产物类型是否已选择对应格式规范。
- Markdown 公式是否遵守项目渲染规则。
- LaTeX、PPT、图表、实验记录和审计报告是否可追溯。
- 文献分类、文献边界和 BibTeX 是否可追溯。
- 强 claim 是否已经通过相应门禁。
- 未完成证明、实验异常和文献边界不清是否进入研究债务队列。
