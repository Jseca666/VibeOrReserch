# 门禁协议

## 1. 假设账本门禁

每个命题必须明确：

- 空间与拓扑。
- 支撑集。
- 紧性或矩条件。
- 映射连续性或可测性。
- Wasserstein 阶数和地面距离。
- 对偶化或最优化所需条件。

## 2. 反例优先门禁

证明前必须检查：

- 边界例。
- 退化权重。
- 空纤维。
- 非紧支撑。
- 不连续映射。
- 闭包缺失。
- \(W_1/W_2\) 混用。

## 3. 证明审计门禁

审计结论只能是：

- `通过`
- `有条件通过`
- `不通过`

若为 `有条件通过`，必须列出具体条件。

## 4. 运筹可解性门禁

任何模型或风险约束必须回答：

- 是否凸。
- 是否可有限维化。
- 是否可对偶化。
- 是否能嵌入路径/调度/资源约束。
- 算法复杂度是否可接受。

## 5. 论文 claim 审批门禁

论文强主张必须标注：

- `定理已证`
- `条件成立`
- `数值支持`
- `建模假设`
- `文献已有`
- `猜想`

## 6. Markdown 公式渲染门禁

所有项目产物必须遵守 `08_agents工作流/产物输出格式规范.md`；数学 Markdown 文档同时遵守 `08_agents工作流/输出文档格式规范.md`。

交付前重点检查：

- 普通正文中没有裸单美元数学公式。
- `$$` 行间公式分隔符成对出现。
- 代码块和命令示例中的美元符号没有被误改。
