# EvoScientist 轻量对齐审计说明

本文件用于审计当前项目的自进化科研工作流与 EvoScientist 思想之间的对应关系。它不是完整 EvoScientist 平台集成方案，也不替代 `自进化自动工作闭环协议.md`、`研究债务队列.md`、`下一轮任务包队列.md`、`evolution_reports/`、Auto-Gate 门禁裁决或最终对外责任确认。

## 1. 对齐目标

当前项目已经形成：

```text
输入触发 -> T1-T7 分类 -> 角色调用 -> 产物验收 -> RD 债务 -> EVO report -> NXT 任务包
```

本文件补充说明：这些机制如何映射到 EvoScientist 风格的记忆演化，使研究记忆不只是备忘录，而能反过来指导下一轮任务周转。

本项目暂不做：

- 不安装或运行 EvoScientist 平台。
- 不把本项目角色系统改写成 EvoScientist 原生多 agent 平台。
- 不把未审查 claim、未关闭债务或一次性异常写成稳定研究记忆。
- 不让 EvoScientist/evo-memory 替代 Auto-Gate 门禁裁决、claim 审批、研究债务关闭、证明审计或最终对外责任确认。

## 2. 参考机制

本文件采用轻量映射，而不是逐项复制。参考对象包括：

- EvoScientist：多智能体科研系统、持久记忆、科研工作流与自进化机制。
- EvoSkills：`evo-memory`、`research-ideation`、`experiment-pipeline` 等技能说明。
- EvoScientist 论文中的两类记忆和三类演化机制。

其中最相关的抽象是：

| EvoScientist 元素 | 含义 | 本项目采用方式 |
|---|---|---|
| \(M_I\) Ideation Memory | 记录可行方向、失败方向、方向取舍 | 映射到 `07_研究记忆/方向记忆.md` |
| \(M_E\) Experimentation Memory | 记录可复用实验、训练、调试策略 | 拆分映射到 `证明策略记忆.md` 与 `实验记忆.md` |
| IDE | 从研究候选中提炼可持续方向 | 映射到方向发现、主线取舍和 NXT 任务包生成 |
| IVE | 对失败进行分类，区分实现失败与方向失败 | 映射到 RD 队列、反例搜索、证明审计和实验失败记录 |
| ESE | 从成功实验或成功流程中抽象可复用策略 | 映射到证明策略记忆、实验记忆和 EVO report 的记忆写回 |

## 3. 本项目的轻量映射

| EvoScientist 层 | 本项目对应机制 | 说明 |
|---|---|---|
| Researcher Agent | T1 公式整理、T5 文献边界、T7 主线自进化 | 负责发现候选方向、整理理论主线和提出下一轮任务 |
| Engineer Agent | T3 优化模型、T4 小规模验证与实验 | 负责把理论接口落到模型、脚本、验证协议和求解器行为 |
| Evolution Manager / Auto-Gate | 研究债务协调员、EVO report、研究记忆写回、Auto-Gate 系统内裁决 | 负责记录、汇总、触发下一轮，并按保守规则执行系统内推进、暂缓、降级、否决、关闭候选和 NXT 生成；不等同最终对外责任确认 |
| Persistent Memory | `07_研究记忆/` | 只记录跨周期稳定经验，不替代债务和 claim 审批 |

## 4. IDE / IVE / ESE 触发口径

### 4.1 IDE：方向演化

触发情形：

- T1/T5/T7 发现新的候选研究方向、辅助桥梁或主线替代方案。
- 文献边界显示某条方向需要降级、重命名或改写贡献边界。
- NXT 任务包连续两轮指向同一新方向，且该方向不只是一次性整理任务。

写回位置：

- `07_研究记忆/方向记忆.md`

允许写入：

- 可行方向。
- 暂缓方向。
- 失败或需谨慎方向。
- 重新激活条件。

不得写入为稳定方向：

- 未经文献边界审查的新颖性判断。
- 未关闭 `claim阻断` 债务的强 claim。
- 一次对话中的直觉想法。

### 4.2 IVE：失败分类演化

触发情形：

- T2 证明审计不通过。
- T4 小规模验证失败、脚本不可运行或求解器行为异常。
- 反例搜索击穿候选命题。
- 文献边界显示核心 claim 已有或只能降级。
- 同一类 RD 债务连续两轮无法关闭。

失败分类：

| 分类 | 含义 | 写回位置 |
|---|---|---|
| 实现失败 | 脚本、模型编码、big-M、求解器设置或实验协议导致失败，可重试 | `实验记忆.md` 或 RD 队列 |
| 证明缺口 | 命题可能仍可成立，但缺假设、选择定理、闭性、对偶条件或反例预筛 | `证明策略记忆.md` 或 RD 队列 |
| 基础方向失败 | 反例或文献边界表明方向本身不应作为强 claim 推进 | `方向记忆.md` |
| 待判定 | 证据不足，暂不写入稳定记忆 | 只进入 RD 队列或 NXT 任务包 |

不得把 `待判定` 写成稳定失败记忆。

### 4.3 ESE：成功策略演化

触发情形：

- T2 证明策略通过审计，并可复用于其他命题。
- T4 小规模验证脚本运行成功，且形成可复用求解器/数据/日志经验。
- T5 文献边界审查形成可复用的 related work 判断模式。
- 某个 RD 债务关闭后，其关闭过程具有跨周期复用价值。

写回位置：

- `07_研究记忆/证明策略记忆.md`
- `07_研究记忆/实验记忆.md`
- 必要时同步到 `方向记忆.md`

不得把一次性成功写成通用策略；必须记录适用上下文和证据来源。

## 5. 记忆写回判定表

每个关键 EVO report 结束时，应补充或检查以下判定表。若本轮是轻量问答，可写“本轮无记忆写回”。

| 字段 | 判定内容 |
|---|---|
| 是否触发 IDE | 是 / 否；若是，写入方向记忆的候选条目 |
| 是否触发 IVE | 是 / 否；若是，失败分类为实现失败 / 证明缺口 / 基础方向失败 / 待判定 |
| 是否触发 ESE | 是 / 否；若是，写入证明策略记忆或实验记忆的候选条目 |
| 是否只是研究债务 | 是 / 否；若是，只进入 RD/NXT，不写成稳定记忆 |
| 证据来源 | EVO-ID / RD-ID / NXT-ID / 命题卡 / 实验日志 / 文献边界 |
| 写回位置 | 无 / 方向记忆 / 证明策略记忆 / 实验记忆 |
| 是否需要人类确认 | 无 / 工具安装 / 算力投入 / 方向偏好 / 投稿前 / 公开发布前 / 强 claim 写入前 |

## 7. Pro 专项评审问题

交给 Pro 时，建议只问以下问题：

1. 当前 RD/NXT/EVO 是否足以形成有效科研自进化循环，还是仍缺关键闭环字段？
2. `07_研究记忆/` 是否足以承担 EvoScientist 风格的 \(M_I/M_E\)，还是需要单独新增记忆写回规则？
3. 每个 EVO 是否必须增加 IDE/IVE/ESE 触发字段，还是现有“记忆写回”字段已足够？
4. NXT-20260502-006 与 NXT-20260502-007 后，什么结果应写入方向记忆、证明策略记忆或实验记忆？
5. 当前是否需要完整 EvoScientist 平台集成，还是继续保持 Markdown 级轻量映射更合适？
